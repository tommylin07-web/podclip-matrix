# 切片选题：哪一段值得切？

一套可复用的判断标准。人工或 LLM 从整期转写稿挑片段时，按这些信号扫一遍。

## 8 条标准（大致按重要性）

1. **能独立成立**——脱离整期上下文也能听懂，且有一个完整的点。播客里大量是寒暄、过渡、口水话，直接排除。
2. **前三秒有钩子**——开头要有反常识、悬念或冲突，让人刷到时停下。例：「卖军火的改行做杯子」「留学顾问的良心：不把不适合的人送出国」。
3. **要么有情绪，要么有用**——情绪=猎奇/反差/共鸣/感动；有用=涨知识/可操作。两头占一头，中间平淡的不要。
4. **命中平台热点 + 受众**——优先蹭当下热点，且落在高互动赛道（育儿/鸡娃、留学、出海创业、急救科普、原生家庭…）。
5. **有一句能当标题/封面的"金句"**——这句直接变钩子标题和封面大字。没金句的段落传播力弱。
6. **时长 40–60 秒为主**——竖屏短视频甜区。太短信息不足，太长完播率掉。
7. **安全 + 品牌调性**——避开平台敏感题材（凶案、夸张医疗、政治），符合账号气质。
8. **一期内角度不重复**——覆盖不同主题，扩大受众面，而不是 5 条讲同一件事。

## 流程

1. 读带时间戳的逐句稿（whisper.cpp 的 `.srt`/`.json`）。
2. 很多节目开头的"预告金句"或嘉宾的强观点就是现成钩子——以此为线索。
3. 回去定位完整段落，记下起止秒。
4. 为每条配：封面大字钩子（`cover_hook`）、平台标题（`pub_title`）、正文/简介（`desc`）、标签（`tags`）。
5. 写入 `matrix/cuts/ep<N>.jsonl`。

## 局限与改进

- 只看文字稿：听不到语气、笑点、停顿；ASR 有错字 → 是"有依据的判断"，非百分百。
- **数据反馈闭环**：发布一段时间后，用真实播放/点赞反向给各类切片的标准加权——越选越准。建议把平台后台数据回灌，定期复盘"哪类切片真的爆"。
